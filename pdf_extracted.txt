Project  Nexus  is  a  self-hosted,  privacy-first  AI  agent  swarm  designed  to  take  high-level

2.  Core  Objectives  
●  Zero  Cost:  No  paid  APIs  (OpenAI/Anthropic).  ●  Privacy:  Local  execution  via  Ollama/LocalAI.  ●  Autonomy:  Multi-agent  orchestration  (Manager,  Dev,  QA,  Designer).  ●  Persistence:  Skill-based  memory  and  long-term  context  retention.  ●  Connectivity:  Integration  with  messaging  apps  for  remote  status  updates.

3.2  Component  Roles  
1.  Manager  Agent:  Parses  user  input,  creates  the  PRD,  assigns  tasks,  and  oversees  
the
 
workflow.
 2.  Designer  Agent:  Uses  tools  like  Antigravity  or  Excalidraw  (via  CLI/API)  to  draft  
UI/UX.
 3.  Developer  Agents  (FE/BE):  Write  code  based  on  the  PRD.  4.  QA  Agent:  Runs  the  code  in  a  sandbox,  executes  unit  tests,  and  captures  errors.  5.  Memory  Module:  Stores  past  snippets,  successful  patterns,  and  project  context.

5.  Low-Level  Design  (LLD)  
5.1  The  Workflow  Logic  (The  Loop)  
1.  Requirement  Phase:  User  gives  a  prompt.  Manager  Agent  generates  a  Markdown  
PRD.
 2.  Design  Phase:  Designer  Agent  generates  a  UI  schema.  3.  Development  Phase:  ○  Manager  splits  tasks  into  tasks.json.  ○  FE/BE  agents  pull  tasks  and  write  files  to  a  local  directory.  
4.   5.  Verification  Phase:  QA  Agent  triggers  a  docker-compose  up  or  a  test  runner.  6.  Reiteration  Loop:  If  Exit  Code  !=  0,  logs  are  sent  back  to  the  respective  agent  for  a  
fix.
 7.  Human-in-the-loop  (HITL):  Before  the  final  merge,  the  Manager  pings  the  user  via  
Telegram/WhatsApp
 
for
 
approval.
 
5.2  Database  Schema  (ChromaDB)  
●  Collection:  project_context  (Stores  current  PRD,  file  paths,  logic  flow).  ●  Collection:  skills_memory  (Stores  solved  bugs  and  code  snippets).
