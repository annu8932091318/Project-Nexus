# generate-plan-fact-report

Генерирует отчёт план/факт — сравнение плановых и фактических данных проекта.

## Описание

Сравнивает плановые и фактические данные по трём измерениям: сроки (вехи), бюджет (статьи), скоуп (результаты). Используется в фазе закрытия для оценки результатов проекта перед итоговым отчётом.

**Фаза:** 6 — Закрытие
**Результат:** `plan-fact-report.md`

## Триггеры

| Язык | Команды |
|------|---------|
| Русский | «сформируй план-факт», «подготовь отчёт план/факт», «сравни план и факт» |
| English | "generate plan-fact report", "plan vs actual", "variance report" |

## Входные данные

| Данные | Обязательно | Примечание |
|--------|:-----------:|------------|
| План проекта | да | Источник плановых вех, бюджета, результатов |
| Фактические данные | да | Предоставляются пользователем в чате: фактические даты, затраты, статусы |
| Устав проекта | нет | Для списка результатов. Если нет — агент запрашивает у пользователя |
| Тип отчёта | нет | `interim` или `final` (по умолчанию: interim). Влияет только на заголовок |

## Структура результата

Отчёт содержит до трёх разделов: отклонения по срокам (план/факт по каждой вехе с отклонением), отклонения по бюджету (план/факт по статьям, если есть данные), статус результатов (завершено, в работе, не начато, отменено).

## Зависимости

**Требует:** `generate-project-plan` (утверждённый план) + фактические данные от пользователя

**Разблокирует после утверждения:** `generate-closure-report`

## Пример

> Сформируй план-факт. План проекта в знаниях. Фактические данные: М1 завершена 15 апреля (план — 10 апреля), М2 — 20 мая (план — 15 мая), общие расходы $48K из $50K бюджета.
