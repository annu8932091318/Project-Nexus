# generate-charter

Генерирует устав проекта — базовый документ фазы инициации.

## Описание

Устав определяет цель, скоуп, сроки, бюджет, команду, ограничения и ключевые риски проекта. Это первый создаваемый артефакт, на котором основаны все последующие скиллы.

**Фаза:** 1 — Инициация
**Результат:** `project-charter.md`

## Триггеры

| Язык | Команды |
|------|---------|
| Русский | «сформируй устав», «подготовь устав проекта», «создай charter», «нужен устав проекта» |
| English | "generate charter", "create project charter", "build charter", "draft project charter" |

## Входные данные

| Данные | Обязательно | Примечание |
|--------|:-----------:|------------|
| Данные о проекте | да | Бриф, свободное описание, ответы заказчика, неструктурированные заметки |
| Ответы заказчика | нет | Заполненный опросник или свободные ответы |
| Ограничения проекта | нет | Бюджетные, временные, технические, организационные |
| Состав команды | нет | Если есть — заполняется раздел команды; иначе только Спонсор и PM |

**Минимум для старта:** цель проекта + хотя бы одно из: сроки, бюджет, скоуп.

## Структура результата

Устав содержит 8 разделов: цель проекта, скоуп (включено/исключено), сроки и вехи, бюджет, команда и роли, ограничения и допущения, топ-3 риска с мерами реагирования, блок согласования.

## Зависимости

**Требует:** данные о проекте от пользователя (предыдущие скиллы не нужны).

**Разблокирует после утверждения:**

- `generate-risk-register` (режим: initial) — фаза 1
- `generate-project-plan` — фаза 2

## Пример

**Команда:**

> Сформируй устав проекта. Разрабатываем мобильное приложение для трекинга фитнес-целей. Сроки: 4 месяца, бюджет $50K, команда: PM + 2 разработчика + 1 дизайнер + 1 QA.

**Агент выдаёт:** структурированный устав со всеми 8 разделами, заполненными на основе предоставленных данных, с помеченными допущениями, готовый к ревью и утверждению.
