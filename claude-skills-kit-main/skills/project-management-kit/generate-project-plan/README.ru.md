# generate-project-plan

Генерирует план проекта с WBS, вехами, зависимостями и картой ресурсов.

## Описание

План проекта декомпозирует скоуп из устава на рабочие пакеты, устанавливает сроки и зависимости, определяет вехи с критериями завершения и распределяет ресурсы. Основной артефакт планирования.

**Фаза:** 2 — Планирование
**Результат:** `project-plan.md`

## Триггеры

| Язык | Команды |
|------|---------|
| Русский | «сформируй план проекта», «подготовь план», «составь project plan», «нужен план проекта» |
| English | "generate project plan", "create project plan", "build plan", "prepare project plan" |

## Входные данные

| Данные | Обязательно | Примечание |
|--------|:-----------:|------------|
| Утверждённый устав | да | Источник цели, скоупа, сроков, бюджета, команды, ограничений |
| Оценки длительности | нет | Оценки по рабочим пакетам. Если нет — агент оценивает, помечая как допущения |
| Загрузка команды | нет | Доступность по периодам, отпуска, частичная занятость. Если нет — равномерное распределение |

## Структура результата

План содержит: резюме проекта, таблицу фаз и рабочих пакетов (макс. 20 WP), вехи с критериями завершения, матрицу зависимостей, карту ресурсов (% загрузки по периодам), допущения и ограничения.

**Ключевые правила:** единица длительности единообразна (рабочие дни для проектов до 3 мес., недели — свыше); даты не выходят за дедлайн устава (отмечается при невозможности); вехи — контрольные точки, не задачи.

## Зависимости

**Требует:** `generate-charter` (утверждённый устав)

**Разблокирует после утверждения (параллельно):**

- `generate-risk-register` (режим: detailed)
- `generate-comm-plan`

## Пример

> Сформируй план проекта. Устав утверждён и загружен в знания. Оценок длительности нет — оцени по объёму.

Агент создаёт план с оценёнными длительностями (помечены как допущения), распределением ресурсов и отмечает конфликты расписания при их наличии.
