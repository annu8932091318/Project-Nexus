# generate-comm-plan

Генерирует план коммуникаций с матрицей стейкхолдеров, расписанием коммуникаций и правилами эскалации.

## Описание

План коммуникаций распределяет стейкхолдеров по влиянию и интересу, определяет, какую информацию каждый получает, через какой канал и с какой частотой, и устанавливает процедуры эскалации.

**Фаза:** 2 — Планирование
**Результат:** `comm-plan.md`

## Триггеры

| Язык | Команды |
|------|---------|
| Русский | «сформируй план коммуникаций», «подготовь comm-plan», «кому и как коммуницировать» |
| English | "generate comm plan", "create communication plan", "build comm plan" |

## Входные данные

| Данные | Обязательно | Примечание |
|--------|:-----------:|------------|
| Устав проекта | да | Источник стейкхолдеров, целей, скоупа, ограничений |
| План проекта | да | Источник вех, фаз — для привязки коммуникаций к событиям |
| Данные о стейкхолдерах | нет | Дополнительные стейкхолдеры, не указанные в уставе |
| Предпочтения по каналам | нет | Каналы коммуникации команды (Slack, Teams, Email и т.д.) |

## Структура результата

План включает: матрицу анализа стейкхолдеров (квадрант влияние/интерес), матрицу коммуникаций (стейкхолдер, тип информации, частота, канал, ответственный), правила эскалации, шаблоны ключевых коммуникационных событий.

## Зависимости

**Требует:** `generate-charter` + `generate-project-plan` (оба утверждены)

**Разблокирует:** нет зависимых скиллов в MVP.

## Пример

> Сформируй план коммуникаций. Устав и план проекта в знаниях. Для ежедневной коммуникации используем Slack, для встреч — Google Meet.
