# Claude Memory Auditor для Cowork — Аудит и очистка файловой памяти

Находит противоречия, устаревшие записи, дубли и шум во всех слоях памяти Cowork.

---

## Что делает

Memory Auditor для Cowork проверяет файловые слои памяти, специфичные для среды Cowork: auto-memory (MEMORY.md + отдельные файлы), CLAUDE.md, User Preferences и Project Instructions. Выявляет проблемы, которые со временем снижают качество работы Claude, и предлагает конкретные правки только с явным подтверждением пользователя.

Это компаньон скила [memory-auditor-chat](../memory-auditor-chat/), который работает с нативной памятью Claude.ai (Memory Edits, Memory Summary). Используйте **memory-auditor-cowork** при работе в десктопном приложении Claude Cowork.

## Как работает

1. **Определение контекста** — автоматически распознаёт среду Cowork (проект или standalone) и доступные слои памяти
2. **Обзор** — показывает статистику по слоям и количество потенциальных проблем (можно остановиться на этом шаге)
3. **Глубокий анализ** — классифицирует проблемы по четырём категориям: противоречия, устаревшее, дубли, шум
4. **Действия** — для файлов auto-memory: прямое редактирование/удаление; для CLAUDE.md, User Preferences и Project Instructions: пошаговые инструкции для ручного исправления

## Быстрый старт

Откройте сессию Cowork и скажите:

- "аудит памяти"
- "проверь память"
- "почисти память"

Claude проведёт аудит и покажет результаты в структурированном отчёте.

## Ключевые отличия от memory-auditor

| Аспект | memory-auditor | memory-auditor-cowork |
|--------|---------------|----------------------|
| Среда | Claude.ai (чат, проект) | Cowork (standalone, проект) |
| Слои | Memory Edits, Memory Summary, Project Summary, Project Instructions | auto-memory, CLAUDE.md, User Preferences, Project Instructions |
| Инструменты | `memory_user_edits`, `conversation_search` | Read, Write, Edit, Glob, Grep |
| Может изменять | Memory Edits | Файлы auto-memory |
| Формат вывода | Свободный | Фиксированный шаблон |

## Категории проблем

- **Противоречие** — конфликт между записями в разных слоях или внутри одного слоя (высший приоритет)
- **Устаревшее** — факт изменился по данным более поздних записей (высший приоритет)
- **Дубль** — смысловое пересечение записей, которые можно объединить (средний приоритет)
- **Шум** — корректные, но малоценные записи: промежуточные состояния, разовые факты, несоответствия frontmatter (низкий приоритет)

## Требования

- Аккаунт Claude (бесплатный или платный)
- Десктопное приложение Claude Cowork
- Внешние инструменты и API-ключи не требуются

## Безопасность

Memory Auditor никогда не выполняет полный сброс памяти. Каждое изменение файлов auto-memory требует явного подтверждения. Скилл не может напрямую изменять CLAUDE.md, User Preferences и Project Instructions — вместо этого предоставляет пошаговые инструкции. Индекс MEMORY.md всегда поддерживается в консистентном состоянии после изменений.
