# memory-auditor-cowork — Руководство пользователя

Аудит всех слоёв файловой памяти Claude в Cowork: поиск и исправление противоречий, устаревших записей, дублей и шума.

---

## Как начать

Откройте сессию Cowork и используйте одну из фраз:

- "аудит памяти"
- "проверь память"
- "почисти память"
- "что у меня в памяти"

Claude распознает скил и начнёт рабочий процесс.

---

## Пошаговый сценарий

1. **Определение контекста** — Claude автоматически определяет среду Cowork (проект или standalone) и доступные слои памяти: файлы auto-memory, CLAUDE.md, User Preferences, Project Instructions.

2. **Обзор** — Claude показывает структурированную сводку: количество файлов auto-memory (с разбивкой по типам), ключевые блоки CLAUDE.md, наличие User Preferences и Project Instructions. Выводит количество потенциальных проблем. На этом шаге можно остановиться.

3. **Глубокий анализ** — после подтверждения Claude проверяет каждую запись во всех доступных слоях и классифицирует проблемы по четырём категориям: противоречия, устаревшее, дубли, шум. Также проверяет консистентность индекса MEMORY.md (битые ссылки, потерянные файлы).

4. **Действия** — Claude представляет результаты в фиксированном формате отчёта, сгруппированные по приоритету. Для файлов auto-memory — конкретные операции редактирования, удаления, объединения. Для остальных слоёв — пошаговые инструкции для ручного исправления.

5. **Подтверждение** — никакие изменения не происходят без явного одобрения. Можно подтверждать по одному или сказать "применяй всё" (Claude покажет полный список перед исполнением). После каждой операции MEMORY.md обновляется автоматически.

---

## Что проверяется

| Слой | Что проверяет Claude | Может ли Claude исправить? |
|------|---------------------|---------------------------|
| auto-memory | Индекс MEMORY.md, отдельные .md-файлы (frontmatter + содержимое) | Да — редактирование, удаление, объединение |
| CLAUDE.md | Глобальные и проектные инструкции | Нет — предоставляет инструкции |
| User Preferences | Персональные настройки в system prompt | Нет — инструкция для Settings > Profile |
| Project Instructions | Инструкции проекта | Нет — инструкция для настроек проекта |

## Примеры

### Пример 1: Быстрый аудит в проекте Cowork

**Вы пишете:**
> аудит памяти

**Claude отвечает:**
> ### Cowork project Memory Audit
> **Контекст:** Cowork project
>
> | Layer | Status | Entries/Size | Notes |
> |-------|--------|-------------|-------|
> | auto-memory | found | 8 файлов: 3 user, 2 feedback, 2 project, 1 reference | ... |
> | CLAUDE.md | found | 45 строк | рабочее пространство, правила навигации |
> | User Preferences | found | present | язык: русский, стиль: прямой |
> | Project Instructions | found | present | правила проекта, naming conventions |
>
> Найдено 5 потенциальных проблем. Перейти к детальному анализу?

### Пример 2: Исправление проблем auto-memory

**Вы пишете:**
> да, покажи проблемы

**Claude отвечает:**
> | # | Layer | Category | Priority | Issue | Proposed Action |
> |---|-------|----------|----------|-------|-----------------|
> | 1 | auto-memory | Duplicate | Warning | feedback_style.md и user_preferences.md оба содержат "прямой стиль коммуникации" | Объединить в user_preferences.md |
> | 2 | auto-memory | Outdated | Error | project_status.md: "Фаза 1", но Project Instructions: "Фаза 3" | Удалить project_status.md |
>
> **auto-memory (can modify):**
> - [#1] Объединить содержимое feedback_style.md в user_preferences.md, удалить feedback_style.md, обновить MEMORY.md
> - [#2] Удалить project_status.md, убрать из MEMORY.md
>
> Подтвердить действия? (да для всех / да для #N / отклонить #N)

## Советы для лучших результатов

- Запускайте аудит периодически — файлы auto-memory накапливаются со временем, особенно в активных проектах.
- Начинайте с обзора, чтобы оценить масштаб, прежде чем переходить к полному анализу.
- Проверяйте здоровье индекса MEMORY.md — потерянные файлы и битые ссылки часто появляются после ручных правок.
- Используйте "применяй всё" только после просмотра полного списка.

## Частые вопросы

**В: Аудит удалит что-то без моего разрешения?**
О: Нет. Каждое изменение файлов auto-memory требует явного "да" или "применяй". Для остальных слоёв скил только даёт инструкции — вы вносите изменения сами.

**В: Что если у меня ещё не настроена auto-memory?**
О: Скил проверит другие доступные слои (CLAUDE.md, User Preferences, Project Instructions). Если ни один слой не доступен — сообщит об этом и завершит аудит.

**В: Чем отличается от обычного memory-auditor?**
О: Обычный memory-auditor работает с нативной памятью Claude.ai (Memory Edits, Memory Summary) через `memory_user_edits`. Эта версия работает с файловой памятью Cowork (auto-memory, CLAUDE.md) через файловые инструменты (Read, Write, Edit).

**В: Что если я хочу сбросить всю память разом?**
О: Скил откажет в этом запросе. Полный сброс памяти необратим. Вместо этого скил помогает точечно исправить конкретные записи.

## Ограничения

- Не может напрямую редактировать CLAUDE.md, User Preferences и Project Instructions — предоставляет инструкции для ручного исправления.
- Не анализирует нативную память Claude.ai (Memory Edits, Memory Summary) — для этого используйте memory-auditor.
- Не анализирует и не удаляет историю чатов.
- Работает только в средах Cowork (десктопное приложение или Claude Code).
