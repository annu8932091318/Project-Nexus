# decision-log — Руководство пользователя

Извлекает структурированные решения из заметок встреч, Slack-тредов и email-цепочек. Формирует чистый лог решений отдельно от action items — с контекстом, альтернативами и участниками.

---

## Как начать

Откройте диалог с Claude и используйте одну из фраз:

- `«Зафиксируй решения из этой встречи: [текст заметок]»`
- `«Decision log из этих meeting notes»`
- `«Извлеки решения из этого треда Slack»`
- `«Обнови мой decision log решениями из сегодняшнего звонка»`
- `«Залоги решения из этой email-переписки»`

Claude распознает скил и начнёт рабочий процесс.

---

## Пошаговый сценарий

**Шаг 1 — Вставьте текст.** Передайте заметки со встречи, Slack-тред или email-переписку. Формат не важен — подходит любой неструктурированный текст.

**Шаг 2 — Выберите формат (если не указан).** Claude спросит, какой формат предпочтительнее: таблица (одна строка на решение) или карточки (отдельный блок на каждое решение). Если вы уже указали предпочтение — вопрос будет пропущен.

**Шаг 3 — Получите извлечённые решения.** Claude сканирует текст и находит точки принятия решений — моменты, где был сделан выбор, зафиксирована позиция или согласован подход. Action items и информационные утверждения исключаются.

**Шаг 4 — Режим append (опционально).** Если передать существующий decision log вместе с новыми заметками, Claude добавит только новые решения и отметит дубли.

## Примеры

### Пример 1: Новый лог из заметок встречи

**Вы пишете:**
> «Зафиксируй решения: Договорились перенести мобильный релиз на две недели из-за бага аутентификации. Денис предложил оставить веб-релиз по плану — все согласились. Бэкенд-команда переходит на Redis для сессионного кэша.»

**Claude отвечает:**
> Найдено 2 решения. Decision log (формат карточки):
>
> **Решение 1: Перенос мобильного релиза на две недели**
> - Decision: Перенести мобильный релиз на две недели
> - Context: Баг аутентификации не устранён
> - Alternatives: —
> - Participants: —
> - Date: —
> - Related: Баг аутентификации
>
> **Решение 2: Веб-релиз по плану**
> - Decision: Веб-релиз выходит по расписанию
> - Context: Предложение Дениса, команда согласилась
> - Alternatives: Перенос веб-релиза (отклонён)
> - Participants: Денис + команда
> - Date: —
> - Related: —

### Пример 2: Пополнение существующего лога

**Вы пишете:**
> «Обнови мой decision log:
> [существующий лог]
> ---
> Заметки сегодня: Решили перейти на новый API gateway до конца Q2.»

**Claude отвечает:**
> Найдено 1 новое решение. Дублей: 0.
>
> Добавлено в лог:
>
> **Решение 4: Переход на новый API gateway до конца Q2**
> - Decision: Перейти на новый API gateway
> - Context: —
> - Alternatives: —
> - Participants: —
> - Date: —
> - Related: Дедлайн Q2

## Что вы получите

Структурированный decision log в выбранном формате, готовый к вставке в:

- Confluence или Notion
- GitHub Wiki или README
- Папку с проектной документацией
- Отдельный `.md`-файл (используйте шаблон из `templates/decision-log-template.md`)

Каждая запись содержит до шести полей. Поля, не упомянутые в исходном тексте, остаются пустыми — Claude не придумывает информацию.

---

## Советы для лучших результатов

- Чем больше контекста в заметках, тем богаче вывод. Формулировки «решили X, потому что Y» дают более качественные записи, чем просто «решили X».
- В режиме append сначала вставьте существующий лог, затем добавьте разделитель (`---`) перед новыми заметками.
- Если нужна конкретная дата на всех записях — укажите её один раз в сообщении: «Встреча 2026-04-08».
- Используйте формат таблицы для Notion-баз и таблиц. Карточки лучше подходят для Markdown-документов.

---

## Частые вопросы

**В: Claude тоже будет извлекать action items?**
О: Нет. Скил фокусируется исключительно на решениях. Action items — отдельный домен, намеренно исключён.

**В: Что если мои заметки на английском?**
О: Скил определяет язык автоматически и отвечает на том же языке.

**В: Что если в тексте нет решений?**
О: Claude явно сообщит, что решений не найдено, и не будет генерировать пустые записи.

---

## Ограничения

- Только текстовый ввод — аудио, видео и изображения не поддерживаются
- Прямое подключение к Slack, email и таск-трекерам отсутствует — текст вставляется вручную
- Не оценивает качество или корректность решений
- Не даёт рекомендаций и не предлагает альтернативы
- Поля остаются пустыми, если информация отсутствует в исходном тексте
