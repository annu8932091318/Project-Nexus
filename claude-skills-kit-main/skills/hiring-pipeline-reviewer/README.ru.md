# Hiring Pipeline Reviewer

Превращает разрозненные заметки о собеседованиях в структурированный еженедельный отчёт по кандидатам — с флагами, оценками и чёткими следующими шагами.

---

## Обзор

Hiring Pipeline Reviewer генерирует структурированный статус по всем кандидатам в вашем пайплайне найма на основе заметок о собеседованиях и оценочных листов. Скил сводит оценки, выявляет застрявших кандидатов и формирует конкретные рекомендации по каждому — всё в одном отчёте. Используйте этот скил при подготовке к еженедельному синку с рекрутером, при принятии решений о продвижении или отклонении кандидатов, а также после очередного раунда собеседований.

---

## Требования

- Заметки о собеседованиях или оценочные листы (plain text, .md или .txt)
  - Можно вставить прямо в чат или указать путь к файлу
  - Один блок заметок по всем кандидатам или отдельные блоки на каждого
  - Заметки могут быть неструктурированными — скил попытается разделить кандидатов самостоятельно
- Опционально: название позиции, критерии найма, пороговое значение для флага «застрял» (по умолчанию: 5 дней)

**Оптимально:** Заметки по 3–15 кандидатам. Большой пайплайн (20+ кандидатов) может дать объёмный вывод.

---

## Как использовать

1. **Соберите заметки о собеседованиях**
   - Скопируйте заметки из последних собеседований и оценочных листов
   - Формат не важен — заметки могут быть в произвольном виде

2. **Запустите скил, написав:**
   - «Статус по кандидатам»
   - «Ревью пайплайна найма»
   - На английском: «Review hiring pipeline» или «Hiring pipeline status»

3. **Вставьте заметки**
   - Вставьте текст в чат или укажите путь к файлу
   - Опционально укажите название позиции и ключевые критерии найма

4. **Изучите отчёт**
   - Таблица статусов: этап, оценка, следующий шаг
   - Раздел «Флаги»: застрявшие и кандидаты с неполными данными
   - Раздел «Рекомендации»: конкретные действия по каждому
   - Опционально: запросите нарративное обновление для рекрутера

## Примеры

### Пример 1: Еженедельный ревью по одной позиции

**Входные данные:**
```
Алекс Чен — Телефонный скрининг 09.04.2026
Сильная коммуникация, релевантный опыт. Оценка 4/5. Назначить интервью 1.

Джордан Ли — Интервью 1 07.04.2026
Пробелы в системном дизайне. Оценка 2/5. Скорее всего отклонить.

Сэм Ривера — Интервью 2 05.04.2026
Обновлений нет с прошлой недели. Команда разделилась 2/2. Нужно обсудить.

Кейси Парк — Заявка 12.04.2026
Резюме ещё не смотрели.
```

**Результат:**
```markdown
## Статус пайплайна найма — Engineering Manager — 2026-04-16

### Таблица кандидатов

| Кандидат     | Этап               | Последнее действие | Оценка | Следующий шаг         | Флаг         |
|--------------|--------------------|--------------------|--------|----------------------|--------------|
| Алекс Чен    | Телефонный скрининг| 2026-04-09         | 4/5    | Назначить интервью 1 | —            |
| Джордан Ли   | Интервью 1         | 2026-04-07         | 2/5    | Отклонить            | —            |
| Сэм Ривера   | Интервью 2         | 2026-04-05         | —      | Нужен командный синк | ⚠ застрял   |
| Кейси Парк   | Заявка             | 2026-04-12         | —      | Рассмотреть резюме   | ❓ нет данных |

### Флаги
- ⚠ Сэм Ривера — без активности с 2026-04-05, следующий шаг не определён
- ❓ Кейси Парк — нет оценочных данных

### Рекомендации
- Продвинуть: Алекс Чен — высокая оценка, чёткий следующий шаг
- Отклонить: Джордан Ли — существенные технические пробелы
- Нужно решение: Сэм Ривера — команда разделилась; требуется синк перед принятием решения
```

### Пример 2: Многоэтапный пайплайн со смешанными заметками

**Входные данные:**
```
Мария Т. — оффер отправлен 10.04, ждём ответа. Сильный кандидат 5/5.

Бен К. — второе интервью 08.04, небольшие вопросы по культуре, 3.5/5.
Дальше: проверка рекомендаций или отклонение.

Дев С. — пропал после первичного скрининга 01.04, не отвечает.
```

**Результат:**
```markdown
## Статус пайплайна найма — 2026-04-16

### Таблица кандидатов

| Кандидат | Этап           | Последнее действие | Оценка | Следующий шаг              | Флаг       |
|----------|----------------|--------------------|--------|----------------------------|------------|
| Мария Т. | Оффер отправлен| 2026-04-10         | 5/5    | Ждать ответа               | —          |
| Бен К.   | Интервью 2     | 2026-04-08         | 3.5/5  | Проверка рекомендаций      | —          |
| Дев С.   | Скрининг       | 2026-04-01         | —      | Закрыть или пометить неакт.| ⚠ застрял  |

### Флаги
- ⚠ Дев С. — нет ответа с 2026-04-01 (15 дней)

### Рекомендации
- Продвинуть: Мария Т. — оффер ожидает ответа; следить за статусом
- Нужно решение: Бен К. — небольшие вопросы; согласовать с командой перед проверкой рекомендаций
- Отклонить: Дев С. — нет ответа; рекомендуется закрыть
```

## Триггеры

Используйте любую из этих фраз для запуска скила:

| Английский | Русский |
|------------|---------|
| Review hiring pipeline | Ревью пайплайна найма |
| Hiring pipeline status | Статус по кандидатам |
| Summarize candidates | Сводка по кандидатам |
| Weekly candidate update | Еженедельный отчёт по найму |
