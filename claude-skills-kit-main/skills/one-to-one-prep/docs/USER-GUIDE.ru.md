# one-to-one-prep — Руководство пользователя

Готовьтесь к ежемесячным встречам 1-on-1 за секунды: вставьте заметки и данные задач — получите структурированную агенду с трекингом action items, темами для обсуждения и вопросами по мотивации.

---

## Как начать

Откройте диалог с Claude и используйте одну из фраз:

- «подготовь 1-on-1 с [имя]»
- «prep для ван-он-вана»
- «сгенерируй агенду для 1-on-1 с [имя]»
- «подготовь встречу один на один с [имя]»

Claude распознает скил и запросит входные данные.

---

## Пошаговый сценарий

**Шаг 1.** Запустите скил фразой, например: «подготовь 1-on-1 с Анной».

**Шаг 2.** Claude запрашивает заметки с прошлой встречи. Вставьте их обычным текстом — любой формат подойдёт (маркированный список, проза, заметки в своём стиле). Если это первая встреча — скажите об этом: скил переключится в режим онбординга.

**Шаг 3.** Claude запрашивает список задач сотрудника за период с прошлой встречи. Скопируйте напрямую из Jira, Linear, Asana, ClickUp, Notion или любого другого инструмента. Чем больше деталей — тем точнее результат, но даже грубый список работает.

**Шаг 4.** При желании добавьте карьерные цели, IDP или OKR. Если они есть — скил добавит тему по развитию и релевантный вопрос по мотивации.

**Шаг 5.** Claude генерирует prep-документ. Просмотрите его, скопируйте в свои заметки — и вы готовы к встрече.

## Примеры

### Пример 1: Регулярный ежемесячный 1-on-1

**Вы пишете:**
> «Подготовь 1-on-1 с Максимом. Заметки с прошлого: обсудили выгорание, договорились что берёт задачи поменьше. Action item — завершить рефакторинг авторизации до конца месяца. Задачи: AUTH-112 рефакторинг авторизации — Done, FRONT-88 компонент формы — In Progress, FRONT-91 баг с валидацией — открыт 18 дней без обновлений.»

**Claude генерирует:**
> Prep-документ: AUTH-112 — Выполнено; FRONT-91 отмечен как застрявший (18 дней без прогресса); темы обсуждения: [Высокий] застрявший баг и [Высокий] followup по выгоранию; вопросы по мотивации о загрузке и блокере FRONT-91.

### Пример 2: Первая встреча с новым сотрудником

**Вы пишете:**
> «Подготовь 1-on-1 с Игорем. Это первая встреча, заметок нет. Задачи за период: BACK-55 интеграция с платёжным шлюзом — Done, BACK-61 тесты для API — In Progress.»

**Claude генерирует:**
> Prep-документ в режиме онбординга: вместо трекинга action items — вопросы для первого знакомства (ожидания, стиль работы, текущие приоритеты). Раздел ключевых событий анализирует задачи в обычном режиме.

---

## Что вы получите

Выходной документ — Markdown с пятью разделами:

**Action Items с прошлой встречи** — таблица со статусом (Выполнено / В процессе / Просрочено) и комментариями для каждого пункта из предыдущей встречи.

**Ключевые события периода** — структурированная сводка: завершённые задачи, задачи под риском или без прогресса, активные блокеры.

**Темы для обсуждения** — 3–5 тем с приоритизацией: Высокий (блокеры, просрочки), Средний (застрявшие задачи, открытые вопросы), Низкий (достижения, обновления по развитию).

**Вопросы по мотивации и wellbeing** — 3–4 открытых вопроса, выбранных под контекст (загрузка, блокеры, карьерные цели при наличии).

**Шаблон для новых Action Items** — пустая таблица для фиксации договорённостей прямо на встрече.

## Советы для лучших результатов

- Вставляйте список задач напрямую из трекера — даже сырой экспорт обрабатывается хорошо.
- Укажите имя сотрудника — тогда заголовок документа будет корректным.
- Если в периоде не было значимых событий — скажите об этом: скил сосредоточится на вопросах и темах развития.
- Для регулярных 1-on-1 удобно сохранять предыдущий prep-документ как «заметки с прошлой встречи» для следующего запуска.
- Добавляйте OKR ближе к границе квартала — скил автоматически проверит прогресс.

## Частые вопросы

**В: Что если у меня нет заметок с прошлой встречи?**
О: Скажите «это первая встреча» или «заметок с прошлого нет». Скил переключится в режим онбординга и сгенерирует вопросы для первого знакомства вместо трекинга action items.

**В: Скил интегрируется с Jira, Linear или другими трекерами?**
О: Нет — работает только с обычным текстом. Скопируйте список задач из любого инструмента. Никаких API-подключений не нужно.

**В: Можно ли использовать для встреч не ежемесячного формата?**
О: Да. Скил подходит для любых регулярных встреч руководителя с подчинённым — еженедельных, раз в две недели или ежемесячных. Просто вставьте данные за нужный период.

---

## Ограничения

- Не хранит историю встреч между сессиями — каждый запуск независим.
- Не обращается к внешним системам и таск-трекерам напрямую.
- Не пишет performance review и не выставляет оценки.
- Качество результата зависит от полноты входных данных — чем больше контекста, тем точнее prep-документ.
