# feature-guide — Руководство пользователя

Структурированные карточки возможностей Claude по запросу. Спрашивайте на обычном языке — получайте чёткий ответ со статусом, доступностью и вердиктом.

---

## Как начать

Откройте диалог с Claude и используйте одну из фраз:

- «Есть ли у Claude память?»
- «Что такое Projects?»
- «Умеет ли Claude создавать файлы?»
- «Поддерживает ли Claude голос?»
- «Хочу чтобы Claude помнил контекст между сессиями»
- «Scout: custom instructions»

Claude распознает скил и начнёт рабочий процесс.

---

## Пошаговый сценарий

**Шаг 1 — Вы задаёте вопрос о фиче**
Напишите название фичи или опишите, что хотите сделать. Специальный формат не нужен.

**Шаг 2 — Claude определяет фичу**
Если вы назвали фичу — Claude переходит сразу к поиску. Если описали потребность — Claude маппит на ближайшую фичу и при неоднозначности задаёт один уточняющий вопрос.

**Шаг 3 — Поиск актуального статуса**
Claude ищет в официальной документации Anthropic текущий статус, доступность и способ активации.

**Шаг 4 — Карточка фичи**
Вы получаете структурированную карточку со всей ключевой информацией. Дополнительных вопросов не требуется.

**Шаг 5 — Смежные фичи (опционально)**
В конце карточки Claude может предложить 1–2 связанные фичи для дальнейшего исследования.

## Примеры

### Пример 1: Запрос по названию фичи

**Вы пишете:**
> «Что такое Projects в Claude?»

**Claude отвечает:**
> ## Projects
>
> **Что это:** Projects — рабочее пространство в Claude.ai, которое группирует диалоги и даёт Claude постоянную память внутри этого пространства. Подходит для длительной работы, требующей непрерывности.
>
> **Где доступно:** Claude.ai — тарифы Free и Pro
>
> **Как использовать:** Нажмите «Projects» в левой панели на claude.ai, создайте проект и ведите диалоги внутри него. Claude будет помнить контекст предыдущих сессий.
>
> **Ограничения:**
> - Память ограничена одним проектом
> - Не работает в обычных диалогах вне Projects
>
> **Статус:** Доступна
>
> **Вердикт:** Подходит — если нужна постоянная память для текущего проекта, это правильная фича.
>
> Исследовать смежные фичи: Custom Instructions, Knowledge base?

### Пример 2: Запрос через потребность

**Вы пишете:**
> «Хочу чтобы Claude помнил то, что я ему говорил раньше»

**Claude отвечает:**
> Это соответствует двум фичам: **Projects** (постоянная память в рамках рабочего пространства) или **Custom Instructions** (фиксированный контекст, который вставляется в каждый диалог). Какой вариант ближе к вашей задаче — память по проекту или постоянные предпочтения?

---

## Что вы получите

Структурированную карточку фичи на каждый запрос:
- Официальное название фичи
- Описание простым языком
- Доступность по платформам и тарифам
- Пошаговая активация
- Известные ограничения
- Актуальный статус (доступна, бета, только Pro, анонсирована)
- Вердикт о применимости для вашей задачи

---

## Советы для лучших результатов

- Конкретизируйте: «Поддерживает ли Claude голос в мобильном приложении?» — лучше, чем просто «голос»
- Описывайте задачу, если не знаете название фичи: «Хочу чтобы Claude использовал мои файлы как контекст»
- Задавайте вопрос об одной фиче за раз — скил генерирует одну карточку на запрос
- Если web search недоступен, карточка будет помечена как «из данных обучения» — проверьте официальную документацию для актуального статуса

---

## Частые вопросы

**В: Что если фича, о которой я спрашиваю, не существует?**
О: В карточке будет указан Статус: «Не найдено», и Claude предложит 1–2 похожих фичи, которые могут подойти.

**В: Можно ли спрашивать на английском?**
О: Да. Скил определяет язык запроса и отвечает на том же языке. Русский и английский поддерживаются полностью.

**В: Будет ли информация актуальной?**
О: Скил выполняет живой поиск по официальной документации Anthropic. Если web search недоступен, карточка помечается: «Данные из обучения — статус может быть устаревшим».

---

## Ограничения

- Не сравнивает Claude с другими AI-продуктами (ChatGPT, Gemini и др.)
- Не указывает стоимость тарифов — только название плана, где фича доступна
- Не настраивает и не включает фичи за пользователя
- Генерирует одну карточку на запрос
- Для актуального статуса необходим доступ к web search
