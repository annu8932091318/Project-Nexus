# stakeholder-adapter — Руководство пользователя

Превратите один документ в готовые версии для каждой аудитории — без ручного переписывания.

---

## Как начать

Откройте диалог с Claude и используйте одну из фраз:

- «Адаптируй этот документ для руководства и клиента: [вставьте документ]»
- «Нужны разные версии этого статус-апдейта»
- «Сделай версию этого PRD для топов»
- «Подготовь версию для клиента»
- «Адаптируй для стейкхолдеров»

Claude распознает скил и начнёт работу.

---

## Пошаговый процесс

**Шаг 1 — Предоставьте документ**

Вставьте текст документа прямо в сообщение или укажите путь к файлу `.md` или `.txt`. Можно включить документ в триггерное сообщение или вставить позже, когда Claude попросит.

**Шаг 2 — Claude задаёт уточняющие вопросы (если нужно)**

Если вы не указали целевые аудитории, тип документа или ключевое сообщение, Claude задаст их одним запросом:

- Для каких аудиторий нужны версии? (Руководство, Команда, Клиент — одна, две или все три)
- Какой тип документа?
- Есть ли ключевое сообщение, которое должно присутствовать во всех версиях?

**Шаг 3 — Claude генерирует адаптированные версии**

Вы получаете по одному чётко подписанному разделу для каждой выбранной аудитории. Каждая версия готова к копированию и отправке.

**Шаг 4 — Сохранение в файлы (по желанию)**

После показа версий Claude предложит сохранить их как отдельные `.md`-файлы. Скажите «да» — они запишутся в рабочую директорию.

## Примеры

### Пример 1: Статус-апдейт для трёх аудиторий

**Вы пишете:**
> «Адаптируй для руководства, команды и клиента:
> Спринт 12 завершён. Реализован модуль авторизации, исправлены 3 критических бага (AUTH-42, API-17, UI-33). Зависимость по конфигу CDN закрыта DevOps 14 апреля. Осталось: финальные E2E-тесты, нагрузочное тестирование (2 дня). Риск: нагрузочные тесты могут выявить задержки в EU-регионе. Цель поставки: 22 апреля.»

**Что вы получите:**

---
**Версия для Руководства**

К сведению. Спринт 12 завершён. Все критические проблемы решены; инфраструктурная зависимость закрыта 14 апреля. Финальное тестирование идёт по плану — поставка 22 апреля. Один низкоуровневый риск: возможные задержки в EU-регионе под нагрузкой. Ведётся мониторинг; действий не требуется.

---
**Версия для Команды**

Спринт 12 закрыт. Модуль авторизации готов; баги AUTH-42, API-17, UI-33 исправлены. CDN-зависимость: закрыта (DevOps, 14 апр). Осталось: E2E + нагрузочные тесты — 2 дня. Риск: задержки EU под нагрузкой. Ответственный: [назначить]. Дедлайн: 22 апр. Блокеров нет.

---
**Версия для Клиента**

Спринт 12 завершён. Все запланированные функции реализованы, критические проблемы устранены. Идёт финальное тестирование, поставка 22 апреля — в плане. **Следующие шаги:** 20 апреля пришлём ссылку на превью для вашего согласования.

---

### Пример 2: PRD только для руководства

**Вы пишете:**
> «Сделай версию этого PRD для топов: [вставьте PRD]»

Claude генерирует одну версию для руководства — сжатую, с акцентом на решения и бизнес-эффект.

---

## Что вы получаете

- По одному чётко подписанному разделу для каждой аудитории
- Версия для руководства: ~25–35% от оригинала, решение или эффект — в начале
- Версия для команды: полная техническая глубина, структурировано под действие
- Версия для клиента: язык результатов, без жаргона, заканчивается «Следующими шагами»

Если документ содержит потенциально чувствительные внутренние данные, версия для клиента будет помечена ⚠ — вы сами решаете, удалять или перефразировать.

---

## Советы для лучшего результата

- Вставляйте документ **в само триггерное сообщение** — это пропускает шаг с вопросами
- Указывайте аудитории сразу: «для руководства и клиента» — и вопрос Q1 будет пропущен
- Укажите ключевое сообщение, если есть обязательство, дата или решение, которое должно быть во всех версиях
- Проверяйте версию для клиента на наличие внутренних данных перед отправкой — Claude помечает подозрительные места, но не удаляет их автоматически
- Для очень длинных документов (2000+ слов) попросите сжатую версию для руководства (максимум 150 слов)

## Частые вопросы

**В: Можно попросить версию только для одной аудитории?**
О: Да. Напишите «сделай версию для руководства» или выберите только «Руководство» — получите один адаптированный раздел.

**В: Можно попросить скорректировать версию после генерации?**
О: Да. Напишите «сделай версию для руководства короче» или «добавь дату поставки в версию для клиента» — Claude обновит её.

**В: Скил переписывает документ с нуля?**
О: Нет. Он переформатирует и адаптирует существующий контент. Выдуманные факты, даты и обязательства он не добавляет.

**В: Что если документ на английском?**
О: Claude определяет язык источника и адаптирует все версии на том же языке. Если нужен другой язык — укажите явно.

**В: Скил сохранит файлы автоматически?**
О: Нет. Claude спросит после показа версий. Скажите «да» — и файлы сохранятся, или просто скопируйте из чата.

---

## Ограничения

- Адаптирует только существующий контент — не генерирует новую информацию
- Не удаляет чувствительные данные из клиентской версии автоматически — помечает их для вашего контроля
- Не заменяет финальную проверку человеком перед отправкой стейкхолдерам
- Поддерживаемые форматы: вставленный текст, `.md`, `.txt` (не `.docx`, `.pdf`, `.xlsx`)
