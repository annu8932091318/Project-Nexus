# stakeholder-adapter — Один документ для любой аудитории

Перестаньте переписывать один и тот же документ для каждой встречи. `stakeholder-adapter` берёт ваш рабочий документ и создаёт версии для Руководства, Команды и Клиента — каждая настроена под то, что нужно конкретной аудитории.

---

## Что делает скил

Большинство документов пишется для одной аудитории, а потом неловко пересылается всем остальным. Руководство пролистывает технические детали. Инженеры не находят нужное решение в деловом тексте. Клиенты путаются в корпоративных аббревиатурах.

`stakeholder-adapter` решает это, применяя правила адаптации под каждую аудиторию:

- **Руководство / топ-менеджмент** — сначала бизнес-эффект, фокус на решениях, только ключевые вехи, без деталей реализации
- **Команда / инженеры** — полная техническая глубина, структура по задачам и ответственным, конкретные дедлайны, блокеры выделены явно
- **Клиент / внешний партнёр** — язык результатов, без жаргона, чистая проза, завершается конкретными следующими шагами

Исходный контент остаётся нетронутым. Скил переформатирует, а не переписывает.

---

## Как работает

1. Вставьте документ текстом или укажите путь к файлу
2. Выберите, для каких аудиторий нужны версии (одна, две или все три)
3. При желании укажите тип документа и ключевое сообщение, которое должно быть в каждой версии
4. Получите все адаптированные версии в одном ответе — с подписями и готовые к использованию

## Быстрый старт

Откройте диалог с Claude и напишите:

> «Адаптируй этот документ для руководства и клиента: [вставьте документ]»

или

> «Нужны разные версии этого статус-апдейта — для топов, команды и клиента»

Claude проведёт вас через остальное.

---

## Пример

**Исходный документ:** статус-апдейт, написанный для команды разработки, — включает задачи спринта, ID багов и внутренние блокеры.

**После — версия для руководства:**
> **К сведению.**
> Бэкенд второй фазы в плане, поставка 22 апреля. Инфраструктурная зависимость (конфиг CDN) закрыта, блокеров нет. Бюджет в рамках плана. Действий не требуется.

**После — версия для клиента:**
> Мы в графике: поставка запланирована на 22 апреля. Инфраструктура настроена, команда на финальном тестировании. **Следующие шаги:** 19 апреля отправим ссылку на превью для вашего подтверждения.

## Требования

- Аккаунт Claude (бесплатный или платный)
- Claude Cowork (рекомендуется для работы с файлами)
- Внешние инструменты и API не нужны
- Поддерживаемые форматы входных данных: вставленный текст, `.md`, `.txt`

## Для кого

- **Продакт-менеджеры**, которые пишут спецификации и апдейты для разных стейкхолдеров
- **Тимлиды**, готовящие статус-отчёты и для руководства, и для инженеров
- **Проектные менеджеры**, адаптирующие предложения и планы под клиентов и внутренние команды
- **Бизнес-аналитики**, переводящие технические выводы на язык менеджмента или клиента
