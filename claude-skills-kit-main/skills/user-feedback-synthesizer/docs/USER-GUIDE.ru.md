# user-feedback-synthesizer — Руководство пользователя

Преврати сырые feedback-файлы в приоритизированный инсайт-отчёт за один шаг — без тегирования, таблиц и ручной кластеризации.

---

## Быстрый старт

Открой диалог с Claude и напиши:

> «Синтезируй фидбек из [путь к папке или список файлов]»

Claude обработает файлы и сохранит `feedback-insights-YYYY-MM-DD.md` в твою рабочую папку.

---

## Пошаговый процесс

**Шаг 1** — Собери feedback-файлы

Положи все файлы для анализа в одну папку. Поддерживаемые форматы: `.md`, `.txt`, `.csv`.

**Шаг 2** — Запусти скил

Используй одну из фраз:
- «Синтезируй фидбек из interviews/»
- «Разбери пользовательские интервью из /путь/к/файлам»
- «Отчёт по пользовательскому фидбеку из feedback-batch/»

**Шаг 3** — (Опционально) Укажи фокус

Сузь анализ до конкретной темы:
> «Синтезируй фидбек из interviews/ — фокус на опыте оформления заказа»

Если фокус не указан, Claude синтезирует все найденные темы.

**Шаг 4** — Изучи результат

Открой `feedback-insights-YYYY-MM-DD.md`. Ты найдёшь:
- Темы, ранжированные по количеству упоминающих источников
- Цитаты-доказательства по каждой теме
- Таблицу приоритизированных инсайтов
- Открытые вопросы для следующего раунда исследований

---

## Сценарии использования

### Сценарий 1: Ежемесячная партия пользовательских интервью

**Ситуация:** Ты провёл 8 пользовательских интервью за месяц и сохранил каждый транскрипт как `.md`-файл в папке `interviews/march/`.

**Что написать:**
> «Синтезируй фидбек из interviews/march/»

**Что получишь:**
- Темы, извлечённые из всех 8 транскриптов
- Лучшие цитаты по каждой теме со ссылкой на источник
- Таблицу инсайтов: High — подтверждено 3+ интервью, Medium — 2
- Открытые вопросы: сигналы только из одного интервью, которые стоит проверить

---

### Сценарий 2: CSV-экспорт опроса

**Ситуация:** NPS-опрос закончился, ты выгрузил ответы в `nps-q1.csv`. Хочешь понять, что движет неудовлетворённостью.

**Что написать:**
> «Отчёт по пользовательскому фидбеку из nps-q1.csv — фокус на сигналах неудовлетворённости»

**Что получишь:**
- Темы, извлечённые из строк опроса
- Приоритизированные Pain и Confusion-сигналы
- Открытые вопросы для дальнейшего изучения

---

### Сценарий 3: Смешанные источники (интервью + тикеты поддержки)

**Ситуация:** У тебя есть транскрипты интервью в `.md` и выгрузка тикетов в `.csv`. Хочешь сравнить, что пользователи говорят в интервью и с чем обращаются в поддержку.

**Что написать:**
> «Разбери пользовательские интервью — обработай папку interviews/ и tickets-export.csv вместе»

**Что получишь:**
- Единый список тем из обоих источников
- Ссылки на источник в цитатах показывают, какие темы из интервью, а какие из тикетов
- Флаги Split signal там, где интервью и тикеты противоречат друг другу

---

## Советы

- **Больше файлов — лучше паттерны.** При менее чем 3 файлах Claude предупредит, что результаты могут отражать мнение одного человека, а не тренд.
- **Используй фокус для точечного анализа.** Если у тебя 20 файлов, а интересует одна тема, фокус уберёт шум.
- **Цитаты — дословные.** Claude никогда не перефразирует. Если цитата выглядит странно, значит в источнике именно такой текст.
- **Открытые вопросы — твой следующий исследовательский план.** Пункты из этого раздела не достигли порога, но могут быть важны — проверь их до закрытия исследовательского цикла.
