# Retro Pattern Analyzer

Перестаньте читать одни и те же жалобы из спринта в спринт. Выясните, что постоянно возвращается — и что команда давно махнула рукой.

---

## Обзор

Retro Pattern Analyzer читает файлы ретроспектив из нескольких спринтов и формирует структурированный отчёт с повторяющимися болями, нерешёнными action items и стабильными позитивными паттернами. Скил распознаёт любой распространённый формат ретро (Что мешало / Что помогало / Договорённости, Start-Stop-Continue, Plus-Delta и другие) и работает с обычными `.md` или `.txt` файлами — без Jira и внешних интеграций. Используйте этот скил при подготовке к квартальному ретро, выявлении системных проблем, которые не решаются от спринта к спринту, проверке выполнения action items или подготовке отчёта о здоровье команды для руководства.

---

## Требования

- 2 и более файлов ретроспектив в формате `.md` или `.txt`
  - Любой стандартный формат: Что мешало / Что помогало / Договорённости; Start-Stop-Continue; Plus-Delta; свободный текст
  - Файлы можно называть с датами (например, `retro-2026-03-15.md`) — скил определит хронологию автоматически
  - Минимум 2 файла; рекомендуется 3–10 для выявления значимых паттернов
- Не требует внешних инструментов, API или интеграций

**Рекомендация:** файлы из 3+ последовательных спринтов дают наиболее полезные паттерны. Два файла тоже работают, но паттерны будут предварительными.

---

## Как использовать

1. **Соберите файлы ретроспектив**
   - Подготовьте `.md` или `.txt` файлы из последних 3–10 спринтов
   - Разместите в одной папке или укажите пути к файлам
   - Любое форматирование подойдёт — скил определяет секции автоматически

2. **Запустите скил**
   - Скажите: «Проанализируй ретро» или «Паттерны в ретроспективах»
   - По-английски: «Analyze retro files» или «Retro pattern analysis»
   - Укажите путь к папке или список файлов

3. **Скил обрабатывает файлы**
   - Разбирает секции каждого файла (что помогало / что мешало / договорённости)
   - Группирует похожие пункты в темы по всем спринтам
   - Выявляет повторяющиеся боли, нерешённые задачи и позитивные паттерны
   - Рассчитывает частоту и тренд для каждой темы

4. **Изучите отчёт**
   - Получите `retro-patterns-YYYY-MM-DD.md` с тремя блоками: Повторяющиеся боли, Нерешённые action items, Стабильные позитивные паттерны
   - Используйте для открытия следующего ретро, брифинга руководства или приоритизации улучшений

## Примеры

### Пример 1: Квартальный ретро-обзор перед планированием

**Входные данные:**
```
Папка: /sprints/retros/
Файлы: retro-q1-s1.md, retro-q1-s2.md, retro-q1-s3.md, retro-q1-s4.md
```

Каждый файл содержит секции вида:
```
## Что мешало
- Деплой занимал 2+ часа
- Нет ответственного за QA sign-off

## Что помогало
- Ежедневные стендапы держали команду в синхронизации

## Договорённости
- Автоматизировать пайплайн деплоя к спринту 3
```

**Выходные данные (`retro-patterns-2026-04-17.md`):**
```markdown
## 🔴 Повторяющиеся боли (что мешало, ≥2 спринта)

| Тема | Спринты | Частота | Тренд |
|------|---------|---------|-------|
| Медленный / ручной деплой | С1, С2, С3, С4 | 4/4 | ↑ растёт |
| Неясная ответственность за QA | С2, С3 | 2/4 | → стабильно |

## 🔁 Нерешённые action items

| Задача | Поднята в | Повторилась в | Статус |
|--------|-----------|---------------|--------|
| Автоматизировать пайплайн деплоя | С1 | С2, С3, С4 | не решена |

## ✅ Стабильные позитивные паттерны

| Тема | Спринты | Частота |
|------|---------|---------|
| Ежедневные стендапы держат команду в синхронизации | С1, С2, С3, С4 | 4/4 |
```

### Пример 2: Фокус только на проблемах доставки

**Входные данные:**
```
Файлы: retro-sprint-12.md, retro-sprint-13.md, retro-sprint-14.md
Фокус: «доставка и релиз»
```

**Выходные данные:** та же структура отчёта, отфильтрованная по темам, связанным с процессом доставки и выпуска.
