# memory-auditor-chat — Руководство пользователя

Аудит всех слоёв нативной памяти Claude в Claude.ai: поиск и исправление противоречий, устаревших записей, дублей и шума.

---

## Как начать

Откройте диалог с Claude и используйте одну из фраз:

- "аудит памяти"
- "проверь память"
- "почисти память"
- "что у меня в памяти"

Claude распознает скилл и начнёт рабочий процесс.

---

## Пошаговый сценарий

1. **Проверка инструментов и контекста** — Claude проверяет доступность `memory_user_edits`, определяет среду (проект Claude.ai или общий чат) и доступные слои памяти.

2. **Обзор** — Claude показывает структурированную сводку: количество Memory Edits, ключевые темы Memory Summary, наличие Project Summary и Project Instructions. Выводит количество потенциальных проблем. На этом шаге можно остановиться.

3. **Глубокий анализ** — после подтверждения Claude проверяет каждую запись во всех доступных слоях и классифицирует проблемы по четырём категориям: противоречия, устаревшее, дубли, шум. Кросс-слойная верификация выявляет конфликты между Memory Edits, Memory Summary, Project Summary и Project Instructions.

4. **Действия** — Claude представляет результаты в структурированном отчёте, сгруппированном по приоритету. Для Memory Edits — конкретные операции удаления/замены. Для остальных слоёв — пошаговые инструкции для ручного исправления.

5. **Подтверждение** — никакие изменения не происходят без явного одобрения. Можно подтверждать по одному или сказать "применяй всё" (Claude покажет полный список перед исполнением).

**Вы пишете:**

>
> **Memory Edits (можно изменить):**
> - [#1] Заменить запись #3. Подтвердить?

## Что вы получите

Структурированный отчёт по фиксированному шаблону: обзор слоёв, таблица проблем по приоритетам (сначала ошибки, затем предупреждения, затем рекомендации), конкретные действия для Memory Edits (удаление/замена с подтверждением), пошаговые инструкции для слоёв, которые нельзя редактировать программно.

---

## Советы для лучших результатов

- Запускайте аудит периодически — память накапливает шум со временем, особенно в активных проектах.
- Начинайте с обзора, чтобы оценить масштаб, прежде чем переходить к полному анализу.
- Используйте "применяй всё" только после просмотра полного списка — это экономит время, но убедитесь, что каждое изменение корректно.
- В проекте проверяются все четыре слоя. В общем чате — только Memory Edits и Memory Summary.

## Частые вопросы

**В: Аудит удалит что-то без моего разрешения?**
О: Нет. Каждое изменение Memory Edits требует явного "да" или "применяй всё". Для остальных слоёв скилл только даёт инструкции — вы вносите изменения сами.

**В: Можно запустить аудит в общем чате (без проекта)?**
О: Да. Аудит охватит глобальные Memory Edits и Memory Summary. Проектные слои будут недоступны.

**В: Что если я хочу сбросить всю память разом?**
О: Скилл откажет в этом запросе. Полный сброс памяти необратим. Вместо этого скилл помогает точечно исправить конкретные записи.

**В: Что будет, если `memory_user_edits` недоступен?**
О: Скилл автоматически переключится в режим только чтение. Он проанализирует все видимые слои и предоставит ручные инструкции для исправлений вместо автоматического применения.

---

## Ограничения

- Не может напрямую редактировать Memory Summary, Project Summary и Project Instructions — предоставляет инструкции для ручного исправления.
- Лимит 5 запросов `conversation_search` за один аудит.
- Работает только с нативными слоями памяти Claude.ai — не работает с файловой auto-memory, CLAUDE.md или локальными файлами (для этого используйте memory-auditor-cowork).
- Не удаляет чаты — только помечает с рекомендациями.
