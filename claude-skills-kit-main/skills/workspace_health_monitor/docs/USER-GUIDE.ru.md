# workspace_health_monitor — Руководство пользователя

Аудит рабочего пространства менеджера: найди забытые задачи, осиротевшие файлы
и конфликты в заметках, планах и логах — в одном структурированном отчёте.

---

## Как начать

Открой диалог с Claude и используй одну из фраз:

- «Аудит воркспейса — вот мои заметки за месяц»
- «Найди забытые задачи в моих файлах»
- «Проверь рабочую папку на осиротевшие файлы»
- «Еженедельная гигиена workspace»
- «Что я забыл сделать из заметок со встреч?»

Claude распознает скил и начнёт рабочий процесс.

---

## Пошаговый сценарий

**Шаг 1 — Предоставить файлы**
Claude спросит, как ты хочешь передать файлы. Варианты:
- В Cowork: открой доступ к папке — Claude читает файлы напрямую
- Загрузи файлы в диалог
- Вставь содержимое файлов текстом в чат

**Шаг 2 — Назвать активные проекты (опционально)**
Claude может спросить: «Перечисли текущие активные проекты — достаточно названий».
Это помогает точнее выявить осиротевшие файлы. Шаг можно пропустить — Claude выведет проекты из файлов.

**Шаг 3 — Claude сканирует и анализирует**
Claude читает все предоставленные файлы и проводит четыре анализа:
- Какие файлы не связаны ни с одним активным проектом
- Какие action items из заметок так и не были перенесены в таск-трекер
- Какие файлы дублируют или противоречат друг другу
- Какой общий уровень захламлённости

**Шаг 4 — Получить отчёт**
Claude формирует структурированный отчёт из пяти разделов с итоговой оценкой захламлённости.

## Примеры

### Пример 1: Еженедельный аудит заметок

**Ты пишешь:**
> «Вот заметки встреч за последние 3 недели. Запусти аудит воркспейса.»

**Claude отвечает:**
> «Проанализировал 12 файлов. Отчёт об аудите:
> **Оценка: Умеренно** — найдены несколько забытых обязательств.
> Осиротевшие файлы (2): 'планирование-Q1-черновик.md', 'идеи-для-оффсайта.md' — не связаны с активными проектами.
> Забытые action items (4): 'Уточнить с Димой статус бюджета' (встреча-2026-03-18.md), ...
> Дубликаты (1): 'roadmap-v2.md' и 'roadmap-final.md' — почти идентичное содержимое, рекомендую оставить более новый.
> Рекомендации: ...»

### Пример 2: Быстрая проверка перед квартальным ревью

**Ты пишешь:**
> «Перед Q2-планированием проверь workspace. Активные проекты: Discovery Phase, Onboarding Redesign, Q2 OKR.»

**Claude отвечает:**
> «Запускаю аудит относительно 3 активных проектов...
> Оценка: Чисто — workspace в хорошей форме.
> Осиротевший файл (1): 'конкурентный-анализ-янв.md' — вероятно, от завершённой инициативы.
> Забытых action items не обнаружено.
> Рекомендую архивировать файл конкурентного анализа.»

---

## Что ты получишь

Структурированный отчёт об аудите, содержащий:

- **Осиротевшие файлы** — название, последняя дата упоминания, причина флага
- **Забытые action items** — точная формулировка, источник (файл), дата если есть
- **Дубликаты и конфликты** — какие файлы пересекаются, какой вероятно актуальнее
- **Рекомендации по чистке** — приоритизированные: Критично (обязательства с именами) → Важно (конфликты) → Желательно (архивация)
- **Оценка захламлённости** — Чисто / Умеренно / Захламлено / Хаос с кратким резюме

---

## Советы для лучших результатов

- Чем полнее файлы — тем точнее аудит; не нужно фильтровать перед подачей
- Включай файлы за последние 2–4 недели — это даёт наиболее полезные результаты
- Если явно перечислишь активные проекты, точность определения осиротевших файлов сильно возрастёт
- Запускай еженедельно — 10–15 минут аудита экономят часы путаницы
- После аудита попроси: «Помоги перенести найденные action items в формат для Jira/Notion»

---

## Частые вопросы

**В: А если файлов мало — скил всё равно полезен?**
О: Да. Даже 3–5 файлов могут содержать забытые action items или конфликты. Скил адаптируется к объёму входа.

**В: Может ли Claude удалять или перемещать файлы после аудита?**
О: Нет. Скил выдаёт только рекомендации. Все операции с файлами — твоё решение.

**В: Что если все мои файлы привязаны к активным проектам?**
О: Claude сообщит «осиротевших файлов не найдено» и сосредоточится на action items и конфликтах. Чистый результат тоже полезен.

---

## Ограничения

- Не подключается к Jira, Asana, Linear и другим таск-трекерам
- Не анализирует бинарные файлы (изображения, архивы) без текстового содержимого
- При >50 файлах Claude приоритизирует по дате и типу файла
- Качество обнаружения зависит от полноты содержимого файлов
- Не сохраняет состояние файлов между сессиями — каждый раз с нуля
