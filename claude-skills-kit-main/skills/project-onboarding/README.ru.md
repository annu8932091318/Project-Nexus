# project-onboarding

Полный онбординг нового проекта в Claude Cowork за одну сессию.

Генерирует полноценную рабочую среду: `context.md`, правила папки, карту файлов и стартовые шаблоны промптов — на основе интерактивного интервью и автоматического сканирования папки.

## Проблема

Настройка нового проекта в Claude Cowork требует вручную создавать контекстные файлы, писать правила, строить карту файловой структуры и готовить шаблоны промптов. Это повторяемая работа, которую легко сделать непоследовательно.

## Решение

Один скилл, который закрывает весь онбординг:

1. **Интерактивное интервью** — фиксирует цели, структуру, ограничения и участников проекта
2. **Автоматическая карта файлов** — сканирует папку проекта и генерирует структурированный индекс файлов
3. **Правила папки** — генерирует правила проекта по типу (аналитика, контент, бизнес, личный)
4. **Стартовые промпты** — создаёт переиспользуемые шаблоны промптов под тип проекта

## Режимы

| Режим | Триггеры | Результат |
|-------|----------|-----------|
| `new` | «создай проект», «новый проект», «настрой проект» / "create project", "new project", "set up project" | Полное интервью → context.md + folder-instructions.md + карта файлов + промпты |
| `quick` | «быстрый контекст», «создай контекст проекта» / "quick context", "create project context" | Блоки 1–3 + автоскан → context.md |
| `scan` | «обнови карту файлов», «пересканируй» / "update file map", "rescan files" | Только обновление карты файлов → патчит существующий context.md |

## Выходные файлы

```
project-folder/
├── context.md                  # Контекст проекта с автоматической картой файлов
├── folder-instructions.md      # Правила под тип проекта
└── resources/
    └── prompts/
        ├── analyze-data.md     # Пример для аналитических проектов
        ├── write-post.md       # Пример для контентных проектов
        └── ...
```

## Установка

### Claude.ai (Skills)

1. Скачай `project-onboarding.zip` из [Releases](../../releases)
2. Перейди в **Settings → Capabilities → Skills**
3. Загрузи zip-файл
4. Скилл появится в списке Skills

### Claude Cowork / Claude Code

Скопируй папку скилла в личную директорию:

```bash
cp -r project-onboarding ~/.claude/skills/project-onboarding
```

Или установи из этого репозитория:

```bash
git clone https://github.com/KirKruglov/claude-skills-kit.git
cp -r claude-skills-kit/project-onboarding ~/.claude/skills/
```

## Структура скилла

```
project-onboarding/
├── SKILL.md                        # Основные инструкции скилла (английский)
└── resources/
    ├── context-template-ru.md      # Шаблон context.md — русский
    ├── context-template-en.md      # Шаблон context.md — английский
    ├── rules-templates-ru.md       # Наборы правил по типу проекта — русский
    ├── rules-templates-en.md       # Наборы правил по типу проекта — английский
    ├── prompt-templates-ru.md      # Стартовые промпты по типу проекта — русский
    └── prompt-templates-en.md      # Стартовые промпты по типу проекта — английский
```

## Поддерживаемые типы проектов

| Тип | Фокус правил | Стартовые промпты |
|-----|-------------|-------------------|
| Аналитика | Целостность данных, источники, структура выводов | `analyze-data.md`, `compare-sources.md` |
| Контент | Голос бренда, работа с черновиками, адаптация под платформы | `write-post.md`, `adapt-post.md` |
| Бизнес | Трекинг гипотез, лог решений, коммуникация со стейкхолдерами | `research-topic.md`, `prepare-summary.md` |
| Личный | Минимум правил, скорость в приоритете | Без промптов (слишком разнородно — используй `prompt-builder`) |

## Общие правила (включены во все типы)

- **File discovery** — Claude находит файлы по описанию, без необходимости указывать точный путь
- **Защита входных данных** — файлы из input/ только для чтения, без изменений
- **Язык и стиль** — определяется автоматически по языку запроса (русский или английский); прямой стиль, без неспрошенных предложений

## Примеры использования

**Полный онбординг:**
```
> создай проект
```
Claude проводит полное интервью (8 блоков), сканирует папку, генерирует все файлы.

**Быстрый контекст:**
```
> быстрый контекст
```
3 блока вопросов + автосканирование папки → минимальный `context.md`.

**Обновить карту файлов после добавления новых файлов:**
```
> пересканируй файлы
```
Повторно сканирует папку и обновляет секцию карты файлов в существующем `context.md`.

## Язык

Скилл поддерживает **русский и английский** нативно.

- `SKILL.md` — английский (инструкции и триггеры скилла)
- Шаблоны в `resources/` — доступны в версиях `-ru` и `-en`

При запуске скилл определяет язык запроса и автоматически выбирает соответствующий набор шаблонов. Все генерируемые файлы соответствуют определённому языку.

## Связанные скиллы
