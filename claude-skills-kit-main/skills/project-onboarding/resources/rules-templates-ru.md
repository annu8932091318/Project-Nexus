# Шаблоны правил по типу проекта

Загружается модулем Rules (шаг 5). Claude выбирает секцию по типу проекта из ответа 2, затем добавляет общие правила.

---

## Тип: аналитика

```markdown
## Правила проекта

### Работа с данными
- Все расчёты сопровождать указанием источника данных
- Не изменять входные файлы — работать с копиями
- При обнаружении аномалий в данных — сообщить перед продолжением

### Структура выводов
- Формат: факт → интерпретация → рекомендация
- Числа округлять до 2 знаков после запятой если не указано иное
- Сравнения оформлять таблицей

### Выходные файлы
- Результаты анализа сохранять в output/
- Промежуточные расчёты — в drafts/
```

---

## Тип: контент

```markdown
## Правила проекта

### Создание контента
- Следовать brand-voice из context.md (если описан)
- Перед финализацией проверить: тон, длина, структура
- Не публиковать без явного подтверждения пользователя

### Файлы
- Черновики сохранять в drafts/
- Финальные версии — в output/
- Каждый пост/статья — отдельный файл с датой в имени

### Адаптация
- При адаптации между платформами — учитывать формат и аудиторию каждой
- Не копировать текст дословно между платформами
```

---

## Тип: бизнес

```markdown
## Правила проекта

### Исследования и гипотезы
- Все гипотезы помечать как [гипотеза] до валидации
- Ссылаться на источники данных и документов
- Решения фиксировать с датой и обоснованием

### Документация
- Ключевые решения записывать в decisions-log секцию context.md
- Встречи и обсуждения — в notes/
- Финальные документы — в output/

### Коммуникация
- Материалы для стейкхолдеров — без технического жаргона
- Презентации — с executive summary на первом слайде
```

---

## Тип: личный

```markdown
## Правила проекта

### Общее
- Приоритет — скорость и практичность
- Минимум формальностей в оформлении
- Результаты сохранять в output/
```

---

## Общие правила (добавляются ко всем типам)

```markdown
## Общие правила

### File discovery
Если задача ссылается на файл по описанию, теме или типу — а не по точному имени — выполни `ls` или `find` по папке проекта и найди подходящий файл самостоятельно. Не спрашивай уточнение, если можешь определить файл по контексту задачи. Если несколько файлов подходят — покажи список и спроси кратко.

### Язык и стиль
- Язык общения: русский
- Стиль: прямой, структурный, без воды
- Не предлагать следующие шаги без запроса

### Безопасность
- Не удалять файлы без явного подтверждения
- Не изменять входные данные (input/) — только читать
```
