# Шаблоны стартовых промптов по типу проекта

Загружается модулем Starter Prompts (шаг 6). Claude выбирает секцию по типу проекта, адаптирует под контекст (название, терминология) и сохраняет как отдельные .md файлы в resources/prompts/.

---

## Тип: аналитика

### Файл: analyze-data.md

```markdown
## Роль
Аналитик данных

## Задача
Проанализируй файл из input/ по следующему плану:
1. Общая структура данных: строки, столбцы, типы, пропуски
2. Ключевые метрики и распределения
3. Аномалии и выбросы
4. Выводы и рекомендации

## Входные данные
Файл из папки input/ — определи автоматически по контексту задачи.

## Требования к выводу
- Формат: структурированный текст с таблицами
- Каждый вывод: факт → интерпретация → рекомендация
- Сохранить результат в output/
```

### Файл: compare-sources.md

```markdown
## Роль
Аналитик-исследователь

## Задача
Сравни данные из нескольких файлов в input/:
1. Определи общие и уникальные параметры
2. Найди расхождения и противоречия
3. Построй сравнительную таблицу
4. Сформулируй выводы

## Входные данные
Все релевантные файлы из input/.

## Требования к выводу
- Сравнительная таблица
- Список расхождений с указанием источников
- Итоговый вывод: 3–5 предложений
- Сохранить в output/
```

## Тип: контент

### Файл: write-post.md

```markdown
## Роль
Контент-стратег и копирайтер

## Задача
Напиши пост на тему: [тема]

## Требования
- Платформа: [LinkedIn / Telegram / X]
- Тон: экспертный, без мотивационных клише
- Структура: hook → основная мысль → примеры/данные → вывод
- Длина: 150–250 слов для LinkedIn, 100–150 для Telegram

## Ограничения
- Без эмодзи (если не указано иное)
- Без общих фраз и банальностей
- Сохранить черновик в drafts/
```

### Файл: adapt-post.md

```markdown
## Роль
Контент-редактор

## Задача
Адаптируй пост из файла [источник] для платформы [целевая платформа].

## Правила адаптации
- LinkedIn → Telegram: сократить, убрать CTA, добавить прямоту
- LinkedIn → X: сжать до 280 символов, сохранить ключевую мысль
- Telegram → LinkedIn: расширить, добавить контекст и профессиональный тон

## Выход
- Сохранить в drafts/ с суффиксом платформы
```

## Тип: бизнес

### Файл: research-topic.md

```markdown
## Роль
Бизнес-аналитик / исследователь

## Задача
Проведи структурированный ресёрч по теме: [тема]

## План
1. Определение ключевых вопросов
2. Анализ доступных источников из input/
3. Матрица: факты / гипотезы / пробелы
4. Выводы и следующие шаги

## Требования к выводу
- Все гипотезы помечать [гипотеза]
- Указывать источник каждого факта
- Сохранить в output/
```

### Файл: prepare-summary.md

```markdown
## Роль
Аналитик

## Задача
Подготовь сводку по материалам из input/ на тему: [тема]

## Формат
- Executive summary: 3–5 предложений
- Ключевые факты: таблица или список
- Открытые вопросы: что требует уточнения
- Объём: не более 1 страницы

## Выход
- Сохранить в output/
```

## Тип: личный

Стартовые промпты не генерируются. Слишком разнородные задачи — пользователю лучше создавать промпты через скилл prompt-builder.
