# PRD Review Challenger — AI-адвокат дьявола для продуктовых документов

Найди дыры в PRD раньше, чем это сделает команда.

---

## Что делает скил

`prd-review-challenger` — скил для Claude, который критически разбирает PRD, фича-спецификации и продуктовые решения. Он играет роль «адвоката дьявола»: находит слабые допущения, пропущенные требования, риски реализации и логические дыры — до того, как документ уйдёт в команду или разработку.

В отличие от инструментов, которые суммируют или переписывают PRD, этот скил его оспаривает. Он задаёт неудобные вопросы, которые всё равно зададут разработчики и дизайнеры — только раньше, когда исправления обходятся дешевле.

---

## Как работает

Вставь PRD или фича-спецификацию в диалог с Claude. Скил проводит структурированный разбор по четырём осям:

**1. Слабые места и неявные допущения** — утверждения, принятые как данность, но не обоснованные.

**2. Открытые вопросы** — то, на что PRD не отвечает, но команда обязательно спросит.

**3. Риски реализации и UX** — техническая сложность, зависимости, friction для пользователей.

**4. Альтернативные решения** — 2–3 других способа решить задачу, чтобы выбранное решение было осознанным, а не дефолтным.

Плюс **чеклист полноты** — есть ли в PRD user stories, acceptance criteria, edge cases, метрики успеха, rollback-план и другие обязательные компоненты.

---

## Быстрый старт

1. Установи скил (см. `docs/INSTALL.ru.md`)
2. Открой диалог с Claude
3. Напиши: `«Проверь мой PRD»` и вставь текст документа
4. Получи структурированный ревью за секунды

---

## Для кого

Продакт-менеджеры, которые хотят найти дыры в собственном PRD до peer review. Тимлиды, проверяющие спецификацию перед стартом разработки. Все, кто ревьюит чужой документ и хочет структурированный фреймворк для критики.

---

## Требования

- Аккаунт Claude (бесплатный или платный)
- Claude.ai Projects, Claude Cowork или Claude Code
- Коннекторы и интеграции не нужны

---

## Что скил не делает

- Не пишет и не переписывает PRD (для этого — скил product-management write-spec)
- Не заменяет ревью с реальными стейкхолдерами, разработчиками и дизайнерами
- Не знает специфики конкретного продукта без переданного контекста
