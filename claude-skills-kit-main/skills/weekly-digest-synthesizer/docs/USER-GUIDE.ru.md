# Руководство пользователя: Weekly Digest Synthesizer

Как использовать Weekly Digest Synthesizer для компиляции статус-файлов команды в структурированный еженедельный отчёт.

---

## Быстрый старт

Самый быстрый способ получить дайджест:

1. Разместите файлы `.md` или `.txt` со статусами в одну папку
2. Напишите: «Составь еженедельный дайджест из [путь к папке]»
3. Получите файл `weekly-digest-YYYY-MM-DD.md` с обновлениями по проектам

**Результат:** Структурированный дайджест со сводкой, статусами по проектам, таблицей задач и блокерами.

**Время:** ~3 минуты

---

## Сценарии

### Сценарий 1: Пятничный сбор статусов для тимлида

**Ситуация:**
Вы тимлид, ведущий три параллельных направления. Каждый ответственный оставляет еженедельный статус в общей папке. Ваша задача — скомпилировать всё в один дайджест для директора до конца дня. Вручную читать и форматировать три файла — 20–30 минут. Нужно уложиться в 5.

**Что делать:**

1. Убедитесь, что все статус-файлы в общей папке (например, `~/Documents/weekly-statuses/`)
   - Типичные файлы: `alpha-status.md`, `beta-status.md`, `infra-status.md`
   - Структура необязательна — свободный текст тоже принимается

2. Запустите скил: «Составь еженедельный дайджест из ~/Documents/weekly-statuses/»
   - Скил сканирует папку и обрабатывает все `.md` и `.txt` файлы

3. Проверьте черновик дайджеста
   - **Сводка** — соответствует ли она итогам недели
   - **Проекты** — корректны ли статусы и задачи
   - **Риски и блокеры** — все ли активные блокеры зафиксированы

4. Отправьте дайджест
   - Вставьте содержимое в письмо по статусу или перешлите `.md`-файл
   - При необходимости отредактируйте сводку по тону перед отправкой

**Ожидаемый результат:**

Вы получаете `weekly-digest-2026-04-13.md` с:
- Сводкой в 3 предложения
- Секциями по проектам со статус-иконками (✅ / ⚠️ / 🔴), обновлениями и задачами
- Таблицей сквозных задач с ответственными и сроками
- Списком блокеров со ссылками на исходные файлы

Вы отправляете дайджест директору меньше чем за 5 минут.

**Почему это работает:** Вместо того чтобы открывать три файла и вручную структурировать их содержимое, скил агрегирует всё автоматически. Ваша роль — только проверить, а не строить.

---

### Сценарий 2: Менеджер возвращается после недели отсутствия

**Ситуация:**
Вы продукт-менеджер, которого не было неделю. Команда продолжала работать и оставила заметки в четырёх файлах. Нужно понять, что произошло, какие решения приняты и что требует вашего внимания — не читая сотни строк из разных файлов.

**Что делать:**

1. Найдите недельные заметки команды
   - Файлы могут называться `product.md`, `design.md`, `backend-notes.md`, `qa.md`
   - Неструктурированные заметки тоже подходят — скил извлечёт что сможет

2. Запустите скил: «Еженедельный дайджест» и укажите папку или файлы
   - Пример: «Дайджест из ~/team-notes/ за неделю с 7 апреля»

3. Читайте дайджест сверху вниз
   - **Сводка:** 3–4 предложения — быстрая ориентация после отсутствия
   - **Проекты:** смотрите на иконки ⚠️ и 🔴
   - **Сводная таблица задач:** что ждёт follow-up и кто ответственный
   - **Риски и блокеры:** что нужно разблокировать или принять решение

4. Примите меры по блокерам
   - Для задач «TBD» (без ответственного): назначьте в follow-up
   - Для заблокированных проектов: инициируйте быструю синхронизацию

**Ожидаемый результат:**

Полная картина недели в одном документе. Вы видите:
- Какие проекты двигались, а какие под угрозой
- Какие решения приняты в ваше отсутствие
- Какие задачи ждут ответственного или срока

Вы в курсе всего за 10 минут.

**Почему это работает:** Не нужно читать каждый файл — скил сам выводит самое важное и указывает на пробелы. Статус-иконки и секция блокеров сразу показывают, куда смотреть.

### Сценарий 3: Автоматический дайджест каждое утро понедельника

**Ситуация:**
Вы руководитель операций и хотите получать дайджест каждый понедельник утром без ручных действий. Команда сбрасывает пятничные статусы в общую папку. Нужно, чтобы дайджест генерировался автоматически.

**Что делать:**

1. Настройте папку, куда команда сбрасывает статус-файлы
   - Пример: `~/Documents/Claude/weekly-notes/`

2. Создайте расписание в Claude:
   - Напишите: «Создай еженедельную задачу на понедельник в 9:00 — составь дайджест из ~/Documents/Claude/weekly-notes/ и сохрани результат»
   - Скил настроит автоматизацию через систему расписаний Cowork

3. Каждый понедельник находите готовый дайджест
   - Файл `weekly-digest-YYYY-MM-DD.md` уже в рабочей директории
   - Проверьте и перешлите

**Ожидаемый результат:**

Дайджест появляется каждый понедельник утром без ручного труда. 5 минут на проверку вместо 30 на компиляцию.

**Почему это работает:** Скил совместим с расписанием Cowork — запускается автоматически в назначенное время. Папка — входящий канал, дайджест — итоговый документ.

## Советы

### Совет 1: Добавьте явные ключевые слова статуса в файлы

Скил определяет статус по ключевым словам: «заблокировано», «под угрозой», «в срок», «завершено», «задержка», а также по эмодзи ✅ и 🔴. Если ваши файлы содержат эти маркеры, вывод будет более структурированным. Даже одна строка «Статус: заблокировано» в начале файла существенно улучшает качество.

**Совет профи:** Договоритесь с командой об общем формате. Даже простая строка `## Статус: заблокировано` в каждом файле резко повышает качество дайджеста.

### Совет 2: Один файл на проект — лучшая группировка

Скил группирует контент по названию проекта. Если в одном файле несколько проектов, они сгруппируются под именем файла, а не по проектам. Для чистого вывода: один файл — один проект или направление.

**Совет профи:** Называйте файлы понятно: `project-alpha.md`, `design-sprint.md`, `hiring.md` — имя файла становится названием проекта, если заголовок H1 отсутствует.

### Совет 3: Явно фиксируйте блокеры для их выявления

Скил выносит блокеры в отдельную секцию. Чтобы блокер был обнаружен, используйте чёткие формулировки: «Блокер:», «Заблокировано», «Ожидаем», «Зависимость:». Неявные блокеры (спрятанные в абзаце) могут быть пропущены.

**Совет профи:** Если секция «Риски и блокеры» пуста, но блокеры есть — проверьте, явно ли они указаны в исходных файлах. Добавьте строку «Блокер:» и перезапустите.
