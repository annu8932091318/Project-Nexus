# Weekly Digest Synthesizer

Превращайте разрозненные статус-файлы в структурированный еженедельный дайджест за несколько минут.

---

## Обзор

Weekly Digest Synthesizer компилирует статус-обновления из нескольких файлов `.md` и `.txt` в единый структурированный еженедельный дайджест. Скил группирует контент по проектам, выявляет задачи и блокеры по всем файлам и формирует готовый к отправке markdown-отчёт. Используйте этот скил, когда нужно собрать еженедельные статусы команды, подготовить пятничный отчёт для руководства или скомпилировать обновления по нескольким направлениям перед общим совещанием.

---

## Требования

- Один или несколько файлов `.md` или `.txt` со статус-обновлениями
  - Поддерживается любая структура: маркированные списки, свободный текст, форматированные отчёты
  - Файлы должны быть доступны в рабочей директории или по указанному пути
- Дополнительных инструментов или скилов не требуется

**Лучший результат:** файлы с явными названиями проектов в заголовках и ключевыми словами статуса («заблокировано», «в срок», «под угрозой») дают наиболее структурированный дайджест. Файлы со свободным текстом тоже обрабатываются, но вывод будет менее формализован.

---

## Как использовать

1. **Подготовьте статус-файлы**
   - Соберите файлы `.md` или `.txt` с еженедельными обновлениями — своими или командными
   - Разместите их в одной папке или запомните пути

2. **Запустите скил**
   - Напишите: «Составь еженедельный дайджест» или «Скомпилируй статусы из файлов»
   - По-английски: «Compile weekly digest» или «Weekly status digest»
   - Укажите путь к папке или список файлов

3. **Скил обрабатывает файлы**
   - Читает все `.md` и `.txt` файлы из указанного места
   - Извлекает обновления, задачи, ответственных, дедлайны и блокеры
   - Группирует по проекту и выявляет сквозные элементы

4. **Получите дайджест**
   - Файл `weekly-digest-YYYY-MM-DD.md`: сводка, секции по проектам, таблица задач, риски
   - Отправьте команде или вставьте в письмо по статусу

## Примеры

### Пример 1: Тимлид собирает пятничный статус

**Входные данные:** три файла — `project-alpha.md`, `project-beta.md`, `infra.md`

```
# project-alpha.md
## Статус: В срок
- API-интеграция завершена
- Code review пройден, готов к QA
- Задача: Мария — написать тест-кейсы до 17.04.2026

# project-beta.md
## Статус: Под угрозой
- Дизайн задерживается — ждём согласования от клиента
- Блокер: клиент не отвечает с понедельника
- Задача: Алекс — связаться с клиентом до 15.04.2026

# infra.md
## Статус: В срок
- Миграция серверов завершена
- Блокеров нет
```

**Действие:** Скил читает все три файла, извлекает статусы, обновления, задачи и блокер.

**Результат:**
```markdown
# Еженедельный дайджест — 2026-04-13

**Скомпилировано из:** 3 файлов | **Проектов:** 3 | **Задач:** 2

## Сводка

Два проекта в срок; Project Beta под угрозой из-за ожидания согласования от клиента. Один блокер требует follow-up на этой неделе.

## Проекты

### Project Alpha
- **Статус:** ✅ В срок
- **Обновления:** API-интеграция завершена; code review пройден; готов к QA
- **Задачи:** Мария — написать тест-кейсы до 2026-04-17
- **Блокеры:** Нет

### Project Beta
- **Статус:** ⚠️ Под угрозой
- **Обновления:** Дизайн задерживается
- **Задачи:** Алекс — связаться с клиентом до 2026-04-15
- **Блокеры:** Клиент не отвечает с понедельника

### Инфраструктура
- **Статус:** ✅ В срок
- **Обновления:** Миграция серверов завершена
- **Блокеры:** Нет

## Сводная таблица задач

| Ответственный | Задача | Срок | Источник |
|---------------|--------|------|----------|
| Мария | Написать тест-кейсы | 2026-04-17 | project-alpha.md |
| Алекс | Связаться с клиентом | 2026-04-15 | project-beta.md |

## Риски и блокеры
- Клиент не отвечает — project-beta.md
```

### Пример 2: Менеджер обрабатывает неструктурированные заметки

**Входные данные:** два файла со свободным текстом — `team-updates.md`, `q2-initiatives.md`

```
# team-updates.md
Аня завершила редизайн онбординга. Ждём проверки юристами.
Бэкенд исправил баг с авторизацией, который был на прошлой неделе.

# q2-initiatives.md
OKR-дашборд: 60% готов. Блокеров нет.
Найм: 2 интервью на следующей неделе. Решение не принято.
```

**Действие:** Скил извлекает прогресс, определяет статус по тексту; часть задач без ответственного.

**Результат:**
```markdown
# Еженедельный дайджест — 2026-04-13

**Скомпилировано из:** 2 файлов | **Проектов:** 2 | **Задач:** 1

## Сводка

Онбординг готов, баг устранён. OKR-дашборд на 60%, найм идёт. Одна юридическая зависимость ожидает решения.

## Проекты

### Обновления команды
- **Статус:** ✅ В срок
- **Обновления:** Редизайн онбординга завершён; баг авторизации устранён
- **Задачи:** TBD — проверка юристами (срок не указан)
- **Блокеры:** Ожидание юридической проверки

### Q2 Инициативы
- **Статус:** ✅ В срок
- **Обновления:** OKR-дашборд 60%; 2 интервью назначены
- **Блокеры:** Нет
```
